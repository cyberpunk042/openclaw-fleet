# /init

**Type:** Claude Code Built-In Command
**Category:** Configuration
**Available to:** ALL agents (but typically run once during project setup)

## What It Actually Does

Initializes a project for Claude Code. Creates CLAUDE.md (project instructions file) and optionally `.claude/` directory structure. Interactive wizard walks through setting up project-specific rules, conventions, and tool configurations.

In fleet context, /init was used during initial project setup. Agent CLAUDE.md files are now generated by IaC scripts (provision-agent-files.sh) from templates + agent-tooling.yaml, not through /init.

## When Fleet Agents Should Use It

**Fleet agents do NOT use /init.** Project initialization is handled by:
- IaC scripts (provision-agent-files.sh generates per-agent CLAUDE.md)
- Gateway (injects context from 8-layer onion)
- Agent-tooling.yaml (defines per-role configuration)

**One-time use:** /init was used once per project (openclaw-fleet, devops-expert-local-ai, etc.) to bootstrap the initial CLAUDE.md. After that, CLAUDE.md is maintained manually or via scripts.

## Relationships

- CREATES: CLAUDE.md (Layer 3 of 8-layer onion)
- CREATES: .claude/ directory (settings, rules, skills)
- CONNECTS TO: InstructionsLoaded hook (CLAUDE.md parsed at session start)
- CONNECTS TO: /memory (alternative way to edit CLAUDE.md after init)
- CONNECTS TO: provision-agent-files.sh (fleet's approach to CLAUDE.md generation)
- CONNECTS TO: 8-layer onion (CLAUDE.md is Layer 3 — init creates the foundation)
